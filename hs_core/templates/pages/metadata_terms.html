Description of HydroShare Resource Types and Metadata Terms

Generic Resource Type Description

A generic resource is an arbitrary collection of files that are logically grouped together as a resource. There may be any number of files, and files may be of any type. The generic resource type should be used for creating resources for which a formal content data model has not been specified in HydroShare. Generic resources are treated the same way as other resource types in terms of sharing and access control; however, no value added tools are provided given that HydroShare cannot anticipate what the contents of generic resources will be.

Time Series Resource Type Description

The time series is a sequence of individual data values that are ordered in time to record the changing trend of a certain phenomenon. As time series are widely used for hydrologic research and because they are also the primary granule of data sharing within the CUAHSI HIS, the time series will be an important resource type in the HydroShare system. Currently, we define a time series resource such that it can contain only one time series. To define the entities and attributes of a time series, we adopt the definition of a time series as specified by Version 2.0 of the Observations Data Model (ODM2) â€“ see description of a time series result in ODM2 here

ODM2 does not limit the types of time series that can be supported. Any time series supported by the ODM2 information model will be valid for input as a time series resource in HydroShare. The specific implementation of ODM2 that will be used for storing time series in HydroShare is described in more detail in a subsequent section, but, in general, ODM2 describes a time series via the following entities:

  • Site/SamplingFeature: The location at which the time series was observed or generated
  • Person/Organization: The person and organization responsible for creating the time series
  • Method: The procedure used to create the time series
  • Variable/Units: The phenomenon that the time series represents and the units in which the values are recorded
  • Action: The specific action or event that created the time series
  • ProcessingLevel: level of processing or quality control to which the time series has been subjected
  • Offset: The X, Y, and Z location of the time series measurements if offset from the site location
  • DataValues: The data values, their recorded dates/times, and their value-level attributes

Multidimensional (NetCDF) Resource Type Description

Multidimensional space-time data is one of the major data types used for hydrological modeling. For example, the precipitation or surface air temperature data that have two dimensions in space and one dimension in time. As Network Common Data Form (NetCDF) is widely used in hydrological and other geoscience modeling because it contains self-describing metadata and supports the creation of array-oriented datasets which may include three spatial dimensions, a time dimension and other user defined dimensions , we choose NetCDF as the file format to support the sharing of the multidimensional space-time data in HydroShare.

Currently, we define the multidimensional (NetCDF) resource such that it includes only one NetCDF file with ".nc" extension. Any multidimensional space-time data supported by the NetCDF format will be valid for input as a multidimensional (NetCDF) resource in HydroShare. To describe the multidimensional space-time data in NetCDF format, we will use the following entities:

  • Spatial Reference: The coordinate reference system and spatial extent information defined in the NetCDF file.
    • Coordinate reference system: the name of the coordinate reference system.
    • Coordinate reference system representation type:the type of the string text used to express the coordinate reference system. Three types are supported including Proj4 spring, EPSG code, and OGC WKT projection.
    • Coordinate reference system representation text: the string text used to express the coordinate reference system based on the given representation type.
    • North Extent: the north limit of the spatial extent.
    • East Extent: the east limit of the spatial extent.
    • South Extent: the south limit of the spatial extent.
    • West Extent: the west limit of the spatial extent.
    • Extent Unit: the unit of the spatial extent.
  • Variable: The variables defined in the NetCDF file representing the multidimensional space-time data.
    • Name: the defined variable name in file. This name may not explicitly explain what variable it actually represents.
    • Unit: the unit of the variable.
    • Type: the data type of the variable which is upported in the NetCDF enhanced data model.
    • Shape: the shape defined for the variable.
    • Long Name: a descriptive name for the variable to explain what the variable represents.
    • Comment: miscellaneous information about the variable.
    • Missing Value: the data value used to represent missing data value. The data type for missing value should be the same as the variable type.

Geographic Raster Resource Type Description

Geographic raster data are commonly used for representing and managing imagry, digital elevation, and other phenomena which is widely used in hydrological research. The raster can have a single or multiple bands (or layers) of data. A band is represented by a single matrix of cell organized into rows and columns where each contains a value representing information. We choose the GeoTIFF file format to represent the geographic raster data in HydroShare. The GeoTIFF refers to TIFF files which have geographic data embeded as tags within the TIFF file. The geographic data can then be used to position the image in the correct location and geometry on the screen of a geographic information display.

Currently, we define the geographic raster resource such that it includes only one geographic raster in GeoTIFF format which includes a single or multiple bands. Any raster data in GeoTIFF format will be valid for input as a geographic raster resource in HydroShare. To describe the geographic raster data, we will use the following entities:

  • Spatial Reference: The coordinate reference system and spatial extent information defined in GeoTIFF file. The metadata element includes the following sub elements:
  • Band Information:The information to describe the phenomenon represented in each band of the geographic raster data.
  • Cell Information:The information to describe the cell attributes of the band.

Model Program Resource Type Description

A Model Program resource describes the software component of a generic model within the water domain. It aims to fully describe the software-related metadata of a given model program to enables scientists to seamlessly retrieve all the content and information required to get a model up-and-running on their local or server-side machine. This resource consists of uploaded content such as source code, compiled binaries, and documentation, as well as metadata describing the model's software. The Model Program resource can be related to many Model Instance resources (below) to completely describe a simulation and the exact software version used for a particular study, to make data replication possible. The specific metadata elements are as follows:

    {#
  • dc:license: The license for the program and source code (e.g., GPL).
    • #}
  • ModelVersionNumber: Model version and/or build number (e.g., SWAT2012 rev. 629)
  • SupportedOperatingSystem:Compatible operating systems (e.g., Mac OS X version 10.7 and later)
  • ModelReleaseDate: The date that this version of the model was released
  • ModelReleaseNotes:Notes regarding the software release
  • ModelWebpage: A URL to the webpage maintained by the model developers
  • CodeRepository: A URL to the source code repository (e.g., Github, Bitbucket, etc.)
  • ModelUserManual: Uploaded content that describes the general use of the model
  • ModelTheoreticalManual: Uploaded content that describes the theoretical representation of the model
  • SourceCode: Uploaded archive containing the source code of the model

Model Instance Resource Type Description

A Model Instance resource defines the input and output data for a generic hydrological model, for a specific time and place. This resource consists of specific metadata to describe the model content as well as the Model Program resource that is used to execute a simulation. These specific metadata elements are as follows:

  • ModelOutput: A HydroShare class that encapsulates all the metadata elements related to Model Instance resource output files
  • IncludesModelOutput: A boolean value that indicates if the output files are included with the uploaded content
  • ExecutedBy: A HydroShare class that defines the Model Program that executes the current resource
  • ModelProgramName: The name of the model that executes the current resource
  • ModelProgramURL: The HydroShare URL of the Model Program resource that executes the current resource

SWAT Model Instance Resource Type Description

This SWAT Model Instance resource is a specific case of the Model Instance resource that applies to the Soil and Water Assessment Tool. This resource extends the more generic Model Instance metadata terms to accommodate additional SWAT specific terms which have been derived from simulation input files. These specific metadata elements are as follows:

  • ModelObjective:The objective of the SWAT Model (e.g. Hydrology, Water quality, BMPs, and Climate / Landuse Change, etc.)
  • SimulationType:The type of the simulation used (e.g. Normal Simulation, Sensitivity Analysis, and Auto-Calibration)
  • ModelInput:HydroShare Class that includes all the metadata elements related to SWAT Model Instance resource input files
  • warm-upPeriodType:The warm-up period type
  • warm-upPeriodValue:The numeric value of the warm-up period in years
  • rainfallTimeStepType:The type of time step used in the simulation for input rainfall data (e.g. Daily or Sub-hourly)
  • rainfallTimeStepValue:The time step value associated with the rainfall data
  • routingTimeStepType:The type of time step used in the simulation for river routing calculations (e.g. Daily or Hourly)
  • routingTimeStepValue:The time step value used for the river routing calculations
  • simulationTimeStepType:The type of time step type used for model simulation (e.g. Annual, Monthly, Daily, or Hourly)
  • simulationTimeStepValue:The time step value used for simulation
  • watershedArea:The watershed area in Km²
  • numberOfSubbasins:The number of the subbasin within the watershed
  • numberOfHRUs:The number of the Hydrologic Response Units within the watershed
  • DEMResolution:The resolution of the Digital Elevation Model in meters
  • DEMSourceName:The name of the Digital Elevation Model dataset provider
  • DEMSourceURL:The URL of the Digital Elevation Model dataset (if applicable)
  • landUseDataSourceName:The name for the Landuse dataset provider
  • landUseDataSourceURL:The URL for the Landuse dataset (if applicable)
  • soilDataSourceName:The name for Soil dataset provider
  • soilDataSourceURL:The URL for Soil dataset (if applicable)
  • ModelMethod:HydroShare class that includes all the metadata elements related to SWAT model methods used in simulation
  • runoffCalculationMethod:The runoff calculation method used
  • flowRoutingMethod:The flow routing method used
  • PETEstimationMethod:The Potential EvapoTranspiration estimation method used
  • ModelParameter:The parameters used in the model (e.g. Crop rotation, Title drainage , Point source, Fertilizer, Tilage operation, Inlet of draining watershed, Irrigation operation, etc.)
