Description of HydroShare Resource Types and Metadata Terms

Generic Resource Type Description

A generic resource is an arbitrary collection of files that are logically grouped together as a resource. There may be any number of files, and files may be of any type. The generic resource type should be used for creating resources for which a formal content data model has not been specified in HydroShare. Generic resources are treated the same way as other resource types in terms of sharing and access control; however, no value added tools are provided given that HydroShare cannot anticipate what the contents of generic resources will be.

Time Series Resource Type Description

The time series is a sequence of individual data values that are ordered in time to record the changing trend of a certain phenomenon. As time series are widely used for hydrologic research and because they are also the primary granule of data sharing within the CUAHSI HIS, the time series will be an important resource type in the HydroShare system. Currently, we define a time series resource such that it can contain only one time series. To define the entities and attributes of a time series, we adopt the definition of a time series as specified by Version 2.0 of the Observations Data Model (ODM2) â€“ see description of a time series result in ODM2 here

ODM2 does not limit the types of time series that can be supported. Any time series supported by the ODM2 information model will be valid for input as a time series resource in HydroShare. The specific implementation of ODM2 that will be used for storing time series in HydroShare is described in more detail in a subsequent section, but, in general, ODM2 describes a time series via the following entities:

  • Site/SamplingFeature: The location at which the time series was observed or generated
  • Person/Organization: The person and organization responsible for creating the time series
  • Method: The procedure used to create the time series
  • Variable/Units: The phenomenon that the time series represents and the units in which the values are recorded
  • Action: The specific action or event that created the time series
  • ProcessingLevel: level of processing or quality control to which the time series has been subjected
  • Offset: The X, Y, and Z location of the time series measurements if offset from the site location
  • DataValues: The data values, their recorded dates/times, and their value-level attributes

Multidimensional (NetCDF) Resource Type Description

Multidimensional space-time data is one of the major data types used for hydrological modeling. For example, the precipitation or surface air temperature data that have two dimensions in space and one dimension in time. As Network Common Data Form (NetCDF) is widely used in hydrological and other geoscience modeling because it contains self-describing metadata and supports the creation of array-oriented datasets which may include three spatial dimensions, a time dimension and other user defined dimensions , we choose NetCDF as the file format to support the sharing of the multidimensional space-time data in HydroShare.

Currently, we define the multidimensional (NetCDF) resource such that it includes only one NetCDF file with ".nc" extension. Any multidimensional space-time data supported by the NetCDF format will be valid for input as a multidimensional (NetCDF) resource in HydroShare. To describe the multidimensional space-time data in NetCDF format, we will use the following entities:

  • Spatial Reference: The coordinate reference system and spatial extent information defined in the NetCDF file.
  • Variable: The variables defined in the NetCDF file representing the multidimensional space-time data.

Geographic Raster Resource Type Description

Geographic raster data are commonly used for representing and managing imagry, digital elevation, and other phenomena which is widely used in hydrological research. The raster can have a single or multiple bands (or layers) of data. A band is represented by a single matrix of cell organized into rows and columns where each contains a value representing information. We choose the GeoTIFF file format to represent the geographic raster data in HydroShare. The GeoTIFF refers to TIFF files which have geographic data embeded as tags within the TIFF file. The geographic data can then be used to position the image in the correct location and geometry on the screen of a geographic information display.

Currently, we define the geographic raster resource such that it includes only one geographic raster in GeoTIFF format which includes a single or multiple bands. Any raster data in GeoTIFF format will be valid for input as a geographic raster resource in HydroShare. To describe the geographic raster data, we will use the following entities:

  • Spatial Reference: The coordinate reference system and spatial extent information defined in GeoTIFF file.
  • Band Information:The information to describe the phenomenon represented in each band of the geographic raster data.
  • Cell Information:The information to describe the cell attributes of the band.

Model Program Resource Type Description

A Model Program resource type is the executable file containing the model's logic and computational engine. The Model Program resource type consists of the source code or compiled software for executing the model in addition to any documentation related to the model (e.g. user manual and theoretical manual). Beyond the Dublin Core terms, this resource type has the following metadata elements to adequately describe the model:

  • dc:license: The license for the program and source code (e.g., GPL).
  • hsterms:ModelVersionNumber: Model version including a build number (e.g., SWAT2012 rev. 629).
  • hsterms:ModelProgramLanguage: Programming language used to create model (e.g., C++).
  • hsterms:SupportedOperatingSystem:Compatible operating systems (e.g., Mac OS X version 10.7 and later).
  • hsterms:ModelReleaseDate: Date that the model was released.
  • hsterms:ModelReleaseNotes:Release notes for Model Program resource.
  • hsterms:ModelWebpage: URL for Model Program resource webpage maintained by the model developers.
  • hsterms:CodeRepository: URL for project page in a code repository (e.g., Github, Bitbucket, Google Code, etc.).
  • hsterms:hasModelUserManual: Documentation that describes the use of the model.
  • hsterm:hasModelTheoreticalManual: Documentation that describes the processes represented within the model.
  • hsterms:hasSourceCode: The source code used to build the Model Program.

Model Instance Resource Type Description

A Model Instance resource type is the data input and output files for a general hydrological model run for a specific time and place. In addition to the Dublin Core metadata elements, this resource type has specific metadata elements to describe the file content and the relationship between the Model Program and Model Instance resource types. These specific metadata elements are as follows:

  • hsterms:ModelOutput: HydroShare Class that includes all the metadata elements related to Model Instance resource output files.
    • hsterms:ModelOutput / hsterms:IncludesModelOutput: This metadata element is used to show if the output files are included with the uploaded input files or not.
  • hsterms:ExecutedBy: HydroShare Class that includes the information about the Model Program resource that is used to execute the Model Instance resource.
    • hsterms:ExecutedBy / hsterms:ModelProgramName: For this metadata element, there is a drop-down list of the available Model Program resources in the HydroShare system.The user chooses the Model Program resource to be executed using the Model Instance resource files.
    • hsterms:ExecutedBy / hsterms:ModelProgramURL:
      • The URL of the choosen Model Program resource in HydroShare.

SWAT Model Instance Resource Type Description

SWAT Model Instance resource type is a specific Model Instance resource type for a specific hydrological model (SWAT). It is an extension of the Model Instance resource type. The SWAT Model Instance resource type is implemented by extending the Model Instance resource type metadata resource to accommodate metadata elements required to specifically describe the SWAT model and its input files. These specific metadata elements are as follows:

  • hsterms:ModelObjective:The objective of building this SWAT Model Instance resource. This objective could be one or more of the following control vocabularies: Hydrology, Water quality, BMPs, and Climate / Landuse Change. The user can also add a custom objective.
  • hsterms:SimulationType:The type of the simulation used in the SWAT Model Instance resource type. This could be one of the following controlled vocabularies: Normal Simulation, Sensitivity Analysis, and Auto-Calibration.
  • hsterms:ModelInput:HydroShare Class that includes all the metadata elements related to SWAT Model Instance resource input files
    • hsterms:ModelInput / hsterms:warm-upPeriodType:The SWAT Model warm-up period type. Always in years.
    • hsterms:ModelInput / hsterms:warm-upPeriodValue:The SWAT Model warm-up period value in years.
    • hsterms:ModelInput / hsterms:rainfallTimeStepType:The time step type used within the rainfall data in the SWAT Model Instance resource input files. This metadata element has control vocabularies: Daily, and Sub-hourly.
    • hsterms:ModelInput / hsterms:rainfallTimeStepValue:The time step value used within the rainfall data in the SWAT Model Instance resource input files.
    • hsterms:ModelInput / hsterms:routingTimeStepType:The time step type used within the model routing in the SWAT Model Instance resource input files. This metadata element has control vocabularies: Daily, and Hourly.
    • hsterms:ModelInput / hsterms:routingTimeStepValue:The time step value used within the model routing in the SWAT Model Instance resource input files.
    • hsterms:ModelInput / hsterms:simulationTimeStepType:The time step type used for simulating the SWAT Model Instance resource. This metadata element has control vocabularies: Annual, Monthly, Daily, and Hourly.
    • hsterms:ModelInput / hsterms:simulationTimeStepValue:The time step value used for simulating the SWAT Model Instance resource.
    • hsterms:ModelInput / hsterms:watershedArea:The watershed area in Km² in the SWAT Model Instance resource input files.
    • hsterms:ModelInput / hsterms:numberOfSubbasins:The number of the subbasin within the watershed in the SWAT Model Instance resource input files.
    • hsterms:ModelInput / hsterms:numberOfHRUs:The number of the Hydrologic Response Units within the watershed in the SWAT Model Instance resource input files.
    • hsterms:ModelInput / hsterms:DEMResolution:The resolution of the Digital Elevation Model in meters in the SWAT Model Instance resouce input files.
    • hsterms:ModelInput / hsterms:DEMSourceName:The source name for the Digitial Elevation Model data.
    • hsterms:ModelInput / hsterms:DEMSourceURL:The source URL for the Digital Elevation Model data if applicable.
    • hsterms:ModelInput / hsterms:landUseDataSourceName:The source name for the Landuse data.
    • hsterms:ModelInput / hsterms:landUseDataSourceURL:The source URL for the Landuse data if applicable.
    • hsterms:ModelInput / hsterms:soilDataSourceName:The source name for Soil data.
    • hsterms:ModelInput / hsterms:soilDataSourceURL:The source URL for Soil data if applicable.
  • hsterms:ModelMethod:HydroShare class that includes all the metadata elements related to SWAT model methods used in the SWAT Model Instance resource.
    • hsterms:ModeMethod / hsterms:runoffCalculationMethod:The runoff calculation method used in the SWAT Model Instance resource input files.
    • hsterms:ModelMethod / hsterms:flowRoutingMethod:The flow routing method used in the SWAT Model Instance resource input files.
    • hsterms:ModelMethod / hsterms:PETEstimationMethod:The Potential EvapoTranspiration estimation method used in the SWAT Instance resource input files.
  • hsterms:ModelParameter:The parameters used in the model. For SWAT model, there are control vocabularies: Crop rotation, Title drainage , Point source, Fertilizer, Tilage operation, Inlet of draining watershed, Irrigation operation. Also the user can add any other parameters.
